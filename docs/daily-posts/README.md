# docs/daily-posts/ — NAGOYA BITES 日次SNS原稿

毎朝 `/journal-today` を実行すると、この配下に `YYYY-MM-DD.md` が生成されます。
ユーザーはこのファイルをコピペして Note / Instagram / X に**手動投稿**します（コストゼロ運用）。

---

## 運用フロー（所要3-5分/日）

```
1. Claude Code で /journal-today を実行
   ↓
2. journal/drafts/YYYY-MM-DD-slug.html と
   docs/daily-posts/YYYY-MM-DD.md が生成される
   ↓
3. ドラフト HTML をブラウザでプレビュー(python3 -m http.server 8000)
   ↓
4. validator が PASS しているか確認
   ↓
5. journal/drafts/ → journal/ に移動して git push
   ↓
6. docs/daily-posts/YYYY-MM-DD.md を開き:
   - Note セクション → note.com にコピペ投稿
   - Instagram セクション → Instagram アプリにコピペ投稿
   - X セクション → X にスレッドとしてコピペ投稿
```

---

## 各SNSの固有ルール

### Note
- 文字数: 20,000字まで(長文OK)
- 画像: ヘッダー画像を1枚設定すると強い。著作権クリアな素材のみ使用
- タグ: 5-10個。`#名古屋グルメ` `#名古屋ランチ` など広めに。Noteのタグは検索より回遊目的
- 末尾に必ず Journal URL を貼る(SEO的に被リンクになる)
- マガジン: 作成後、1本目からマガジンに入れていくと継続率が可視化される

### Instagram
- キャプション: 2,200字まで。最初の125字が「続きを読む」前に表示される → 要点を先頭に
- ハッシュタグ: 30個まで。`hashtag_pool.json` の bundles に従い8-15個を推奨(つけすぎはスパム判定)
- 投稿タイプ:
  - **フィード**(画像1-10枚)が基本。カルーセル8-10枚はエンゲージメント最大化
  - **リール**(15-60秒動画)を週3本以上 → 10,000フォロワー目標には必須
  - **ストーリーズ**でフィード投稿を必ず24時間以内に再投稿してリーチ拡大
- 最初のコメントにハッシュタグを分離する運用もあり(本文の視認性↑)
- 画像: **転載NG**。自前撮影 or 店舗公式の許可を得た素材のみ

### X
- 1ツイート 280字(日本語140字)以内
- スレッド形式: 1ツイート目に要点 + 👇 の誘導、2ツイート目以降で本論、最終ツイートに Journal URL
- ハッシュタグは1-2個に絞る(つけすぎは逆効果)
- 投稿時刻: 朝7-8時 / 昼12時 / 夜19-21時 が名古屋圏の食系で強い

## 外部媒体の店を紹介した場合

Journal に LOCAL_STORES 未収録の店を採用したら、各SNSでも:

1. 本文に店名と情報源(他媒体記事URL)を明記
2. その店の公式 Instagram がある場合は `@mention` する
3. 画像は**転載禁止**。自前で撮るか、店側から許可を得たもののみ
4. Journal記事の pending_stores.json に登録済みであることを確認

## 10,000フォロワー達成のための補助施策

毎日投稿だけでは届かない。以下を並行実行:

| 施策 | 頻度 | 狙い |
|---|---|---|
| Reels(15-60秒動画) | 週3本 | アルゴリズム優遇枠、リーチ拡大 |
| コラボ投稿(地元シェフ/ブロガー) | 月2回 | 相互フォロー + 信頼性 |
| UGC導線(読者が投稿したくなる診断コンテンツ等) | 常設 | 自然拡散 |
| ストーリーズ質問箱 | 週2-3回 | エンゲージメント率維持 |
| 他媒体引用元への感謝メンション | 記事作成時 | 業界ネットワーク醸成 |

**中間KPI**: 月+1,500人ペース / エンゲージメント率3%以上維持。

## 失敗パターン(踏んではいけない地雷)

- 同じ構文・同じハッシュタグを毎日コピペ → アルゴリズム的に評価ダウン
- 外部媒体の画像転載 → 著作権クレームで即アカウント凍結リスク
- 閉店済み店舗の紹介 → `scripts/audit_journal.js` を月次で必ず走らせる
- オーナー名・個人名の記載 → 匿名運営規約(EDT-001)違反
- フォロワー買い・相互フォロー依頼bot → エンゲージメント率が下がって逆効果

## セッション引き継ぎのために

この運用は毎朝の Claude Code セッションで行うため、**セッション間の記憶はありません**。
引き継ぎに必要な情報はすべて以下に永続化されています:

- `data/journal_queue.json` — 向こう14日のテーマ枠
- `data/journal_published.json` — 過去の公開ログ
- `data/pending_stores.json` — 新規取り込み待ちの店
- `data/editorial_column_backlog.json` — コラムストック
- `agents/editor.md` — 日次運用章

新しいセッションで `/journal-today` を実行すれば、Editor が上記をすべて読んで
当日の記事をドラフトします。
