## 運用ルール

1. **毎週月曜朝に取得**: Monday タスクとして定例化。Orchestrator の FULL モードに組み込む
2. **3週連続で悪化した指標があれば P1 課題として agent-backlog.md に登録**
3. **季節イベント前（GW・忘年会・母の日 等）は該当KWの順位推移を重点監視**
4. **数値だけでなく「なぜそうなったか」の仮説をセットで記録する**
5. **ベースラインが取れる最初の4週間は Phase 0（数字蓄積期）として扱う**
