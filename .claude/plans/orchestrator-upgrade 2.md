# Orchestrator 最高責任者アップグレード計画

## 現状の問題

現在のOrchestratorは「技術的サイト運用マネージャー」にすぎない。
管理範囲がINSPECT/BUILD/DATA/EMERGENCYのみで、**事業成長に責任を持つ最高責任者**としては以下が完全に欠落している：

| 欠落領域 | 具体的に何がない |
|---------|---------------|
| マーケティング | SEO戦略、SNS運用、コンテンツマーケ、トラフィック成長施策 |
| ブランディング | ブランドアイデンティティ、ポジショニング、競合との差別化 |
| 販売促進 | レビュワー獲得、ユーザーエンゲージメント、リピーター施策 |
| マネタイズ | 収益モデル、広告戦略、アフィリエイト、パートナーシップ |
| 成長戦略 | KPI定義・追跡、成長ロードマップ、データ分析 |

## 改善計画

### 1. 新エージェント2名を追加

**Marketer** (`agents/marketer.md`)
- SEO戦略の立案と実行（キーワード最適化、メタ情報、構造化データ）
- SNSコンテンツの企画・原稿作成（Instagram、X等）
- コンテンツマーケティング（特集記事、季節コンテンツ、エリア特集）
- 競合分析・ポジショニング提案
- サイト内CTA最適化・CVR改善

**Strategist** (`agents/strategist.md`)
- ブランディング戦略（名古屋限定×飲食業界運営の強み最大化）
- マネタイズ戦略（広告、アフィリエイト、飲食店向け有料掲載、パートナーシップ）
- レビュワー・ライター獲得施策
- KPI設計と成長ロードマップ
- ユーザーエンゲージメント・リピーター施策

### 2. Orchestratorに新モード3つ追加

| 新モード | トリガー例 | 内容 |
|---------|-----------|------|
| **MARKETING モード** | 「SEO改善して」「SNS投稿作って」「集客施策考えて」 | Marketerに指示、成果物をレビュー |
| **STRATEGY モード** | 「マネタイズどうする？」「ブランド強化して」「成長計画作って」 | Strategistに指示、戦略レビュー |
| **GROWTH モード** | 「レビュワー増やしたい」「KPI確認して」「全体の成長状況は？」 | Marketer + Strategist両方を起動、横断的施策 |

### 3. CLAUDE.md のエージェント構成を更新

```
Orchestrator（最高責任者）
├── [技術部門]
│   ├── Inspector      ← サイト監査・品質管理
│   ├── Builder        ← 実装・デプロイ
│   └── DataKeeper     ← データパイプライン
│
├── [事業部門]  ← NEW
│   ├── Marketer       ← マーケティング・SEO・SNS・コンテンツ
│   └── Strategist     ← ブランディング・マネタイズ・成長戦略
```

### 4. Orchestratorの「意図分類表」を拡張

既存6モード + 新規3モード = 9モード体制

### 5. 報告フォーマットに事業KPIセクション追加

```
### 事業KPI（参考値）
- 想定月間PV: XXX
- SEOスコア: XX/100
- コンテンツ充実度: XX%
- マネタイズ進捗: [ステータス]
```
